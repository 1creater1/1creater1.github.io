三个重点

作为一名学习计算机的学生，我认为想长远的走好每一步，要把握三个重点。

  • 把握到 “T” 字学习结构模型；

  • 把握国际上计算机教学最新阶段的目标；

  • 把握解构和建构两者的关系。

“T”字学习模型

学习计算机，全面的对计算机有一个整体的认识，是T的一横，这一步至少能扩宽自己认识域的边界，尽可能帮助我们俯瞰到计算机领域的全局，而下一步，要结合自身的兴趣，处境，目标来选择某一领域深入发掘，这是T的一竖。随着研究的越深，所触碰到其他知识也越多，以此达到一通百通，触类旁通。

对全局的了解和把控决定了我们的视野有多高，未来的上限会在哪，而对专一领域的学习研究决定了我们的实践能力有多强，未来的下限会在哪。

国际教学第三阶段的目标

从历史来看，国际计算机教学先后经历了三个主要阶段。

  • 第一阶段是上世纪70年代中期至80年代中期，那时理论、技术还不成熟，系统不稳定，因此教学主要围绕若干重要问题讲授不同流派的观点。学生解决实际问题的能力不强。

  • 第二阶段是上世纪80年代中期至本世纪初，当时计算机单机系统的理论和技术已逐步趋于成熟，主流系统稳定，因此教学主要围绕主流系统讲解理论和技术。学生的理论基础扎实，动手能力强。

  • 第三阶段从本世纪初开始，主要背景是随着互联网的兴起，信息技术开始渗透到人类工作和生活的方方面面。因此，技术爆炸迫使教学者必须重构传统的以计算机单机系统为主导的课程体系，其核心内容承担了帮助学生构建专业知识框架的任务，为学生在毕业后相当长时间内的专业发展奠定坚实基础。

现在一般认为问题抽象、系统抽象、数据抽象是计算机类专业毕业生的核心能力。

笔者认为，问题抽象能力的锻炼可以学习数学，算法等知识，系统抽象能力的锻炼可以学习计算机组成原理，操作系统等知识，数据抽象能力锻炼可以学习数据结构等相关知识。

解构和建构

  • 解构，是学习奠定计算机学科的学科知识，包含数据结构、操作系统、计算机网络、计算机组成原理这基础四大件，还有编译原理、数学等相关内容。这一解构，是武林中的内功，是游戏里的英雄等级，是《一人之下》里能拆解万法化为本质的”神明灵“。

  • 建构，是学习用于解决现实的需求问题的技术，也是我们日常用的最多的编程技能，包含像 Java 、C++ 加 Python 这样的编程语言，数据库开发框架、编译器这类技术工具以及设计模式、软件工程这类设计思想等等。这一建构，是武林中的武功秘籍，是游戏里的英雄技能，是《一人之下》里能衍生万法的炁体源流。

解构和建构这两个对子，也符合所谓”阴在阳之中，不在阳之对“，这两者本身并无冲突，想要更好的建构一个项目，必须要解构，了解计算机其中的底层构造。
